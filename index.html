BEM

It is a known fact that the right style guide can significantly increase development speed, debugging, and the implementation of new features in legacy code. Sadly, most CSS codebases are sometimes developed without any structure or naming conventions. This leads to an unmaintainable CSS codebase in the long term. The BEM approach ensures that everyone who participates in the development of a website works with a single codebase and speaks the same language. Using proper naming will prepare you for the changes in the design of the website.

CSS BEM uses compound names for CSS classes, those being “block__element--modifier”. Here’s an example of what a CSS developer writing in the BEM style might look like:

.block
.block__element
.block__element--modifier

Block

Block encapsulates a standalone entity that is meaningful on its own.

  • Naming: Block names may consist of Latin letters, digits, and dashes. To form a CSS class, add a short prefix for namespacing: .block
  • HTML: Any DOM node can be a block if it accepts a class name. <div class ="block">....</div>
  • CSS: Use class name selector only. No tag name or ids. No dependency on other blocks/elements on a page. .block { color : #042; }

Element

An element is a part of a block and has no standalone meaning. Any element is semantically tied to its block.

  • Naming: Element names may consist of Latin letters, digits, dashes and underscores. CSS class is formed as block name plus two underscores plus element name: .block__element
  • HTML: Any DOM node can be a block if it accepts a class name. <div class ="block">....</div>
  • CSS: Use class name selector only. No tag name or ids. No dependency on other blocks/elements on a page. .block { color : #042; }

Modifier

Flags on blocks or elements. Use them to change appearance, behavior or state.

  • Naming: Naming: Modifier names may consist of Latin letters, digits, dashes and underscores. CSS class is formed as block’s or element’s name plus two dashes: .block--mod or .block__elem--mod and .block--color-black with .block--color-red. Spaces in complicated modifiers are replaced by dash.
  • HTML: Modifier is an extra class name which you add to a block/element DOM node. Add modifier classes only to blocks/elements they modify, and keep the original class.
  • CSS: Use modifier class name as selector : .block-- hidden { }. To alter elements based on a block-level modifier : .block--mod .block__elem { } Element modifier : .block__elem--mod { }

Example

.card
.card__image
.card__title
.card__description
.card__btn--primary
.card__btn--secondary

HTML

<div class="card">
  <div class="card__image"></div>
  <h2 class="card__title"></h2>
  <p class="card__description"></p>
  <button class="card__btn card__btn--primary">
  </button>
  <button class="card__btn card_btn--secondary">
  </button>
</div>

SCSS

            
.card {
  // image
  &__image{
    width: 100%;
  }
  // title
  &__title{
    font-size: 2.5rem;
  }
  // description
  &__description{
    font-size: 1.2rem;
  }
  // button
  &__btn{ 
    display: inline-block;
    color: black;
  // primary button
  &--primary{
    background-color: blue;
  }&--secondary{
    background-color: red;
    }
  }
}

CSS

            
.card__image {
  width: 100%;
}
.card__title {
  font-size: 2.5rem;
}
.card__description {
  font-size: 1.2rem;
}
.card__btn {
  display: inline-block;
  color: black;
}
.card__btn--primary {
  background-color: blue;
}
.card__btn--secondary {
  background-color: red;
}
